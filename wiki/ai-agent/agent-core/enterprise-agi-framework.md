# 企业级 AGI 框架

> 原文：SiliconANGLE Breaking Analysis by Dave Vellante & George Gilbert《Forget AGI. The real prize is enterprise AGI》（2026-06-27），中文译本《318| 企业级AGI：AI的真正战场》。

SiliconANGLE 的 Breaking Analysis 团队提出了一个尖锐判断：**前沿模型供应商追逐的"弥赛亚 AGI"可能跑错了方向**——真正的经济奖赏不在于一个无所不能的通用模型，而在于每家企业独有的、可治理的智能系统。

## 核心命题：企业级 AGI vs 弥赛亚 AGI

| | 弥赛亚 AGI（前沿模型供应商追逐的） | 企业级 AGI（真正的奖赏） |
|---|---|---|
| 智能位置 | 集中在一个通用模型内 | 每家企业独有的受治理资产 |
| 方法 | "数据共产主义"——最聪明的人贡献推理轨迹，智能打包进模型分发给所有人 | "数据资本主义"——每家企业控制、治理、推进自身差异化智能 |
| 效果 | 提高所有人的底线，但不创造企业差异化 | 创造企业专有优势，提高上限 |
| 类比 | 通用金融/通用销售/通用医疗知识 | 某家企业实际如何运作的运营模型 |

关键区别：**前沿模型可以提供通用推理，但企业级 AGI 要求每家公司将自己的运营知识作为数据捕捉——并将这些知识视为受治理的、可重用的、复合的资产**。没有这一企业特定层，公司只是在租用竞争对手也能获得的通用智能。

这个对立框架能统领 [[agent-infra-vendor-strategies|各家厂商的 Agent 战略]]：Databricks/Snowflake 从数据平台向上、Palantir 从流程模型、Salesforce/SAP 从记录系统、前沿实验室从参与系统，所有道路都通向智能系统层。

## SoI 四层堆栈模型

企业 AI 软件堆栈分为四层，价值重心在中间的**智能系统（System of Intelligence, SoI）**：

```
┌─────────────────────────────────────────────┐
│  System of Engagement（参与系统）             │  人机交互的新工作界面
│  用户表达意图、澄清歧义、批准行动              │  也是教学界面——教导智能后端
├─────────────────────────────────────────────┤
│  System of Agency（代理系统）                 │  Agent 通过智能系统采取行动
│  回答问题、分析选项、规划行动、操作化决策       │  不脱离企业背景独立行动
├─────────────────────────────────────────────┤
│  System of Intelligence（智能系统 SoI）⭐     │  企业的数字表示——业务状态实时模型
│  连接受治理数据与业务含义、指标、政策、流程     │  企业本体或数字孪生
├─────────────────────────────────────────────┤
│  Data Platform / System of Record（数据平台）  │  告诉企业"发生了什么"
│  存储、交易、事件、记录、文档                  │  必不可少但不解释业务
└─────────────────────────────────────────────┘
```

- **数据平台/记录系统**告诉我们**发生了什么**
- **智能系统**解释**为什么发生、可能发生什么、接下来应该做什么**
- **代理系统**允许智能体通过该智能**采取行动**，而不是绕过它
- **参与系统**成为人、智能体、意图和成果汇聚的新工作界面

**最深刻的洞察：参与系统和智能系统必须协同设计**——客户端教导后端，后端改进客户端。参与层收集用户意图和行为信号，智能层将其转化为受治理背景，代理层随后以更高信心行动。这就是为什么拥有客户端体验在战略上有价值——谁拥有参与层，谁就拥有教导智能后端企业实际运作方式的"行为排气"（behavioral exhaust）的特权访问权。

## 本体成熟度模型（1-9 级）⭐

这是评估任何企业本体/智能系统成熟度的可复用分析工具。三级维度：**数据模型范围与保真度 / 分析复杂度 / 可行动性**。

| 级别 | 名称 | 分析能力 | 可行动性 |
|------|------|---------|---------|
| 1 | Stored Reporting 存储报告 | "这个季度多少交易"——单一系统 | 打印报告，不指导行动 |
| 2 | Enterprise Data Warehouse 企业数仓 | 同一实体多系统未链接记录 | 报告 |
| 3 | Real-Time Event Hub 实时事件中心 | 事件保留上下文可重放 | 带证据的"为什么" |
| 4 | Behavioral Analytics 行为分析 | 相关性 | 人决定 |
| 5 | Predictive Analytics Hub 预测分析中心 | "流失风险是什么，为什么" | 推荐行动，人决定 |
| **6** | **Enterprise Knowledge Graph 企业知识图谱** | 类型化对象、关系、路径推理 | Agent 在狭窄受治理通道按意图行动 |
| **7** | **Semantic Action Layer 语义行动层** | 行动成为模型化数据（前提条件/效果/护栏） | Agent 在护栏内选择并运行行动 |
| 8 | Real-Time Digital Twin 实时数字孪生 | 实时状态=共享事实来源，分析=运营 | Agent 在共享实时状态上协调 |
| 9 | Autonomous Operations Platform 自主运营平台 | 工作流本身成为可编辑数据 | Agent 规划优化，人类设定目标 |

**1-5 级是诊断性的**（改进报告、相关性、行为分析、预测）；**6 级开始向智能体协调迈进**；**7 级及以上是可执行的**。

### Databricks Genie Ontology 当前位置：5-6 级

SiliconANGLE 评估 [[genie-ontology|Databricks Genie Ontology]] 大致处于 **5 到 6 级**——从诊断智能向智能体协调过渡。这是一个相对大多数企业数据环境的强位置，但还不是企业级 AGI 的高端。

**关键判断：从 5-6 级到 7 级及以上的跳跃不能仅通过自下而上推断实现。** 自下而上推断适用于结构（实体、关系、词汇、常用指标），但无法可靠指定运营逻辑（每个流程步骤的条件/结果）和权威规范（政策、法规、合同、禁止事项）。行为显示发生了什么，不能完全定义需要什么。要达到更高层级，本体必须结合更多明确教学、业务流程建模、治理和人在回路验证。

## 描述性本体 vs 可执行本体

这是本体成熟度的核心分水岭：

| | 描述性本体 | 可执行本体 |
|---|-----------|-----------|
| 能力 | 语义协调——就数据含义达成一致 | 表示行动、前提条件、效果、工作流、实时状态、政策约束 |
| 让 Agent | 准确**回答问题** | 自信地**采取行动** |
| Genie Ontology 现状 | ✅ 当前优势 | ❌ 仍在涌现 |

构建可执行本体的三层难度递增：

1. **结构（Structure）** — 业务对象、关系、粒度、共享命名。**自下而上学习最强**。
2. **运营逻辑（Operating Logic）** — 指标计算、每步条件与结果、团队工作流。**难**。
3. **权威与跨领域规范（Authority & Cross-cutting Norms）** — 全公司政策、监管义务、合同约束、要求行动、截止日期、禁止事项。**最难**，不能仅从行为学习。

### 技能提升管道（Promotion Pipeline）

从行为信号到可执行本体的关键转变。纯自下而上=巴别塔（大量本地自动化无共享逻辑），纯自上而下=太慢脆弱。**获胜架构在中间相遇**：

```
01 Author locally（本地编写）
   → 02 Abstract（抽象：LLM 识别重复冲突，使其可读）
   → 03 Route by risk（按风险路由）
   → 04 Govern（治理审查）
   → 05 Promote（提升为共享运营逻辑）
   └─ 且愿意 walk back（回退）/ re-fragment（重新碎片化）
```

技能始于本地知识 → 变得可读 → 与其他技能和政策比较 → 受治理 → 成为智能体可依赖的可重用公司资产。"冰面线是接缝"（the waterline is the seam）。

## 数据层冰面化

> "数据是新的硬件，不是新的石油——必须存在的基础设施，但其价值由其之上的平台解锁。"

企业软件正分为两大类（詹姆斯·邦德"冰面之上/之下"隐喻）：

| | Below the Ice 冰面之下 | Above the Ice 冰面之上 |
|---|----------------------|----------------------|
| 内容 | 数据平台、存储、计算、格式、管道、程序化应用 | 智能系统、代理系统、参与系统 |
| 角色 | 基础设施——必不可少但日益标准化、互换 | 差异化发生处——模型化企业 |
| 定价 | utility pricing 公用事业定价（消费/使用/容量） | outcome pricing 成果定价（基于业务成果） |
| 价值 | 利润压缩 | 定价能力最强 |

随着 Delta/Iceberg 格式趋同、对象存储协议统一，数据基础设施标准化、公用事业化。**新平台层是智能系统（业务流程模型），新应用层是智能体，新定价前沿是业务成果**。客户不应关心底层是 Delta 还是 Iceberg——基础设施应直接工作，这意味着它变得更像硬件。

这对存储/数据基础设施从业者的启示：差异化从性能与成本，向上迁移至语义与治理能力。

## Omnigent 的深层定位：治理 ≠ 学习

[[omnigent-meta-harness|Omnigent]] 让 Databricks 即使在不拥有前端时也能参与——通过 Unity AI Gateway 治理第三方 Agent。但有一个重要局限：

**治理第三方客户端与将客户端与智能系统协同设计不是一回事。** 原生 Genie 体验可以在本体不确定时提出澄清问题、捕捉用户响应、转化为持久企业背景——这是教导 Genie Ontology 的反馈循环。Omnigent 可能能管理和观察外部智能体，但可能无法捕捉原生 Genie 客户端那样丰富的语义反馈。

下一个平台之战不仅是关于治理，更是关于**学习**：谁捕捉到用户的意图、模糊性、纠正和澄清，谁就在改进智能系统方面拥有优势。

### Agent 追踪 = 新型行为排气

智能体追踪正成为一类新的企业数据——类比点击流之于大数据时代。推理轨迹、工具调用、行动路径、失败、重试、中间决策、人类覆盖，将成为驱动智能体诊断、性能改进和系统训练的等效信号。数据量可能比传统应用遥测大几个数量级。**智能体的可观察性可能对 AI 时代变得像点击流对大数据时代一样重要**。

## 通往企业级 AGI 的竞赛

主要供应商阵营从不同位置进入，但所有道路通向同一控制点——智能系统：

| 阵营 | 代表 | 切入位置 |
|------|------|---------|
| 前沿实验室 + Copilot | OpenAI / Anthropic | 参与系统（可能试图进入智能系统） |
| 数据平台 | Databricks / Snowflake | 受治理企业数据，向上移动到业务语义 |
| SaaS 和流程供应商 | Salesforce / SAP / Palantir / Celonis / Blue Yonder / RelationalAI | 记录系统、工作流、领域流程模型（起点更近） |

**关键判断：谁拥有本体，谁就拥有业务定义。** 在 AI 时代，拥有定义比拥有仪表板更重要——如果系统知道权威指标、实体和关系，它就可以动态生成可视化、叙述、应用或智能体工作流。**仪表板成为输出，而不是控制点**。这就是为什么语义层正成为竞争战场（如微软阻止 Databricks Unity 指标在 Power BI 中消费的传闻）。

## 行动项目

不要将 AI 视为模型选择练习，而应视为**企业智能建设项目**。每个高管团队应指派单一负责人，开始构建受治理智能系统——一张动态企业地图，捕捉数据、指标、业务规则、流程逻辑、技能和隐性知识作为智能体可推理和行动的公司资产。选择一个高价值业务领域，建模关键对象，定义权威指标，收集用户编写技能，治理规则，仅在企业有足够背景安全行动处连接智能体。
