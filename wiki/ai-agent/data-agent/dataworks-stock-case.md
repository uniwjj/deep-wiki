# DataWorks 个人量化案例

个人投资者使用 [[dataworks-data-agent|DataWorks Data Agent]] 从零搭建行情分析系统的真实案例。核心问题：**普通人能不能进入高门槛的数据工程场景？**

## 动机

个人投资者不只想看行情软件给出的固定榜单，而是想按自己的问题分析市场。演讲者为百万粉丝 AI 博主、海外 AI 硕士，希望通过 AI Agent 验证"普通人能否进入高门槛数据工程场景"。

具体分析需求：
- 市场活跃度如何？
- 谁真的有资金关注？
- 是趋势增强还是单日异动？
- 自己的候选股票池如何构建？
- A 股与港股跨市场对比

## 传统方式的壁垒

如果不用 DataWorks Data Agent，这不是“写几句 SQL”的问题，而是一整条数据工程链路：

1. 获取、整理、上传大规模原始数据
2. 设计数仓分层与建表语句
3. 编写加工 SQL 与 DataWorks 工作流
4. 配置依赖、调度、监控与质量规则
5. 排查运行错误，再接入 BI 分析工具

**任何一环都可能卡住非专业开发者。**

## 通用 Agent 为什么不够

| 缺失能力 | 说明 |
|---------|------|
| **NEEDS CONTEXT** | 不了解 DataWorks 项目结构、MaxCompute/Hologres 选型 |
| **NEEDS JUDGMENT** | 无法判断引擎选择、表设计合理性 |
| **NEEDS WORKFLOW** | 不理解节点依赖与调度逻辑 |
| **NEEDS OPS** | 不懂 SQL 运行排错、DQC 配置 |

## DataWorks Data Agent 的价值

把专业经验翻译成可执行路径：

- 理解项目、数据表与需求文档
- 创建 DWD、DWS、ADS 分层表
- 生成 SQL 节点和工作流依赖
- 诊断失败节点，辅助修复并重跑
- 配置数据质量规则，接入 ChatBI

## 完整链路（10 步）

### 01. 原始数据获取与处理
A 股 5400+ 支股票（724 万条数据）、港股 2000+ 支（291 万条数据），通过 Python 爬虫获取日线 CSV。先按相同字段合并成宽表（最大约 100 万行），再上传到 MaxCompute（支持单文件 5GB）。

> "本地的 CSV 文件就像桌面上的原材料，MaxCompute 就是一个可以承接大规模加工的后厨。"

### 02. 制作需求文档
先定义问题，才知道数据要被加工成什么。需求文档包含：业务背景、痛点、现状数据量、需求清单。

| 场景 | 分析目标 |
|------|---------|
| Scene 01 | 市场概览 |
| Scene 02 | 活跃股票榜单 |
| Scene 03 | 个股趋势分析 |
| Scene 04 | A 股与港股对比 |

### 03. 发送提示词创建工作流
把需求文档交给 DataWorks Data Agent，要求它按数仓规范建设 A 股与港股日线行情 ADS 层，并同步建设 DWD、DWS 层。

### 04. 理解 DWD/DWS/ADS 分层
用“做饭”的比喻解释数仓分层：

| 分层 | 比喻 | 作用 | 举例 |
|------|------|------|------|
| **DWD** | 洗菜、切菜 | 清洗和标准化明细数据 | 统一市场字段、股票代码、交易日期、OHLCV |
| **DWS** | 加工半成品 | 汇总可复用特征 | 5日/10日/20日均价、成交额排名、涨幅天数 |
| **ADS** | 装盘上桌 | 面向最终分析和查询 | 市场总览看板、A股vs港股对比、活跃股票排名 |

### 05. 节点报错与 Agent 修复（逆风局）
真实运行中，字段类型不对、A 股有港股没有的字段、依赖未配好等问题几乎必然出现。**关键不在于顺风时跑通，在于逆风时能继续**。

> "如果一个产品只能跟我打顺风局，不能打逆风局，那它的价值就有限。"

Agent 能：读取失败日志 → 判断失败原因与影响节点 → 修改 SQL/表结构/配置 → 辅助重新运行。

### 06. 发布生产与结果验证
“上线只是让任务进入生产环境，验收还要看结果表是否符合需求文档预期。”验证项：任务运行状态、结果表非空、最新交易日数据、A 股港股覆盖数量、活跃榜和关键指标。

### 07. 配置数据质量规则（DQC）
“质量规则不是为了炫，而是为了防止数据静悄悄地出错。”

| 规则 | 检测内容 |
|------|---------|
| Check 01 | 表是否为空 |
| Check 02 | 行数异常波动 |
| Check 03 | 日期格式正确 |
| Check 04 | 关键字段非空 |

### 08-10. ChatBI 问答分析 → 闭环
最终不是看表，而是继续追问：

| 问题 | 目的 |
|------|------|
| 哪边市场更活跃？ | 市场热度对比 |
| 哪些股票成交额最高？ | 发现活跃标的 |
| 哪些股票振幅大？ | 识别波动标的 |
| 哪些站上多条均线？ | 技术信号筛选 |
| 哪些适合放入候选观察池？ | 构建个人池子 |

## 对个人投资者的实际帮助

> “不是替你赚钱，而是提高信息处理能力。”

- 自定义分析问题
- 快速筛选活跃股票
- 观察市场热度和波动
- 比较 A 股与港股
- 构建自己的候选股票池

## 产品亮点与感受

- **提高的不只是高手上限，也提高普通人的下限**
- 自然语言 + 需求文档驱动复杂数据开发流程
- 排错、运维、质量监控由 Agent 辅助
- Skill 机制沉淀数据工程经验

## 最终结论

> “三年前要做这套系统，需要一个完整的数据开发团队；现在一个人 + DataAgent 就可以。”

从不会 DataWorks，到完成一套能跑、能监控、能分析的数据系统。**经验可以被产品化、流程化、Skill 化**——写需求文档可做成 Skill、数仓分层可做成 Skill、DQC 配置可做成 Skill，团队经验可互通叠加。复杂系统的门槛正在被往下压。
