## Hermes Agent 凭什么两周封神，自进化还是有点东西！

**文章信息**：
- **原文标题**：Hermes Agent 凭什么两周封神，自进化还是有点东西！
- **来源**：https://www.toutiao.com/article/7626457580352700947
- **作者**：Dukee
- **发表时间**：2026-04-09 02:30

---

**一段话总结**：

Hermes Agent 两周内获 216 个 PR、63 个贡献者，GitHub Stars 突破 35.8k。其核心差异化在于"自进化"：完成复杂任务后自动生成可复用的技能文档，技能在使用中持续迭代优化。三层记忆系统（Session / Persistent / Skill Memory）用 SQLite + FTS5 实现。与 OpenClaw 定位不同——OpenClaw 是多渠道助理平台，Hermes 是自我进化的引擎，两者互补而非对立。

---

**作者判断**：

作者认为 Hermes 代表"智能化思路"——先让机器学会学习，先让 Agent 真正像一个会成长的助手。原文原话："平台可能会变，协议可能会过时，但一个真正懂你、会学习、能自我迭代的 Agent，它的价值是可以穿越周期的。" 但作者也客观指出 OpenClaw 在执行位和生态积累上更强，两者是互补关系。文章末尾推广了作者自己的项目 AllyHub，定位类似。

---

**核心内容**：

**迭代速度惊人**：v0.2.0（3月12日）发布后两周收到 216 个 PR、63 个贡献者。后续版本节奏：3月17日 v0.3.0（流式输出、插件架构），3月底 v0.6.0（MCP 协议），4月3日 v0.7.0（168 个 PR，46 个问题解决）。GitHub Stars 突破 35.8k。

**五个核心差异化**：

1. **闭环学习**：完成复杂任务（通常 5 次以上工具调用）后自动生成 Markdown 技能文档，记录做了什么、怎么做的。下次类似任务直接调用，而非从零推理。

2. **技能自我迭代**：技能文档不是一次性的。发现更好方法时自动更新。Reddit 用户反馈：两小时内自动生成三份技能文档后，重复性研究任务速度提升 40%。

3. **三层记忆系统**：SQLite + FTS5 全文检索 + LLM 摘要。
   - Session Memory：当前对话上下文
   - Persistent Memory：所有历史对话存储和检索
   - Skill Memory：从经验沉淀的可复用流程

4. **API 额度友好**：专门针对记忆架构做了缓存优化，高频调用高阶模型时 Token 账单不会失控膨胀。

5. **契合 Karpathy 的 LLM Wiki 工作流**：底层逻辑与今年 4 月 Karpathy 提出的 LLM Wiki 设计模式完全契合。

**与 OpenClaw 对比（五个维度）**：

| 维度 | OpenClaw | Hermes |
|------|----------|--------|
| 定位 | 多渠道助理平台（Gateway + Control Plane） | 自我进化的引擎（Agent 循环：感知→规划→行动→反思→记忆） |
| 技能来源 | 手动编写 + ClawHub 社区 | 用出来——看你干活，看多了就学会了 |
| 记忆设计 | 代码级编辑（透明可改，需懂技术） | 搜索级检索（问它，它检索、摘要、回答） |
| 安全性 | 曾曝安全风波 | 五层纵深防御（用户授权、命令审批、容器隔离、凭据过滤、注入扫描） |
| 关系 | 当执行位更强 | 当指挥位更合适，提供 `hermes claw migrate` 一键迁移 |
