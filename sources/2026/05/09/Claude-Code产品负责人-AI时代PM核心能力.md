## Claude Code 产品负责人：面试了上百个 PM 后发现，大多数人还在用互联网时代的逻辑应聘 AI 公司

**文章信息**：
- **原文标题**：深度｜Claude Code 产品负责人：面试了上百个 PM 后发现，大多数人还在用互联网时代的逻辑应聘 AI 公司
- **来源**：https://www.toutiao.com/article/7636059353421759014
- **作者**：ZFinance（基于 Lenny's Podcast 对 Cat Wu 的访谈整理）
- **发表时间**：2026-05-04 23:30

---

**一段话总结**：

Anthropic Claude Code 与 CoWork 产品负责人 Cat Wu 在 Lenny's Podcast 中分享 AI 时代产品经理的核心转型：PM 的核心职责从"跨团队对齐"转向"以最快交付为中心"，最重要的能力是在混沌中判断一个月后产品该长什么样。Anthropic 的高速发布靠的不是模型强，而是极简流程 + research preview 机制 + 跨职能团队第二天就能接发布。她面试上百 PM 后的核心观察：代码越来越便宜，产品品味反而越来越贵；未来最有价值的人不是守住岗位说明书的人，而是看到哪里缺人就立刻补位的人。

---

**核心观点（Cat Wu 原话提炼）**：

**1. PM 核心能力转变**
- "前 AI 时代产品规划看 6-12 个月，现在模型能力几周一变，开发周期从半年压缩到一个月、一周甚至一天。PM 核心职责从跨团队对齐转为缩短从构想到交付的周期"
- "稀缺的 PM，是能在混沌里提前看见下一步的人"

**2. 代码便宜，品味变贵**
- "当编写代码的成本骤降，决定写什么才变得更有价值"
- Anthropic 团队"几乎所有 PM 要么以前是工程师，要么现在在 Claude Code 写代码"
- Cat Wu 本人是工程师出身 + VC 经历，加入 Anthropic 后仍在写代码

**3. Anthropic 的组织速度方法论**
- 极简流程：几乎所有功能以 research preview 上线，"告诉用户这是早期想法，未来不一定支持"→ 降低发布心理负担
- 发布流水线：工程师内部验证 → 固定渠道发布 → 文档、营销、DevRel 团队第二天接上
- "我们有严格的指标体系，每周全团队数据复盘，每个人都知道核心目标和趋势"

**4. 岗位边界被打碎**
- "所有角色都在融合——PM 做工程，工程师做 PM 工作，设计师也提交代码"
- 目标：招聘有出色 product taste 的工程师，"可以端到端从看到 Twitter 用户反馈到周末前发布产品"
- "最需要的是能身兼数面、随时换挡、对活儿不分贵贱、一心帮团队加速的人"

**5. 为不完美的 AI 设计产品才难**
- "95% 的自动化不叫自动化——不能 100% 稳定，就不能被真正信任"
- "Agent 拐点不是更会聊天，而是真的能替你干活。用户震撼时刻是 AI 帮你写完代码、做完 PPT、整理完客户简报"

**6. PRD 还在写但有变化**
- "对于特别模糊的功能，写一页纸说明目标、令人愉悦的用例和当前失败模式"
- 基础设施密集型项目仍会写完整 PRD

**7. 人在 AI 时代不可替代的价值**
- 常识和高情商（隐性的利益相关方管理）仍是模型短板
- 面对持续变化需要"向乱而生"的心态：非常狠地排优先级，接受有些事情必须放掉，只要不影响最核心用户价值，可以通过反馈在下一轮修复

**GitHub 爆料**：Claude Code 源码泄露是人为错误——有人用 Claude 辅助写 PR 更新发布包，已加固流程。当事人仍在 Anthropic，团队专注于学习改进。
