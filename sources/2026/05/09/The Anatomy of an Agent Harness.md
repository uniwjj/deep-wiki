## The Anatomy of an Agent Harness（Agent Harness 的剖析）总结

**文章信息**
- **原文标题**：The Anatomy of an Agent Harness
- **来源**：<https://blog.langchain.com/the-anatomy-of-an-agent-harness/>
- **作者**：Vivek Trivedy
- **发表时间**：未明确标注（LangChain 官方博客）

**一段话总结**：本文是 Harness Engineering 领域的定义性文章，由 LangChain 作者 Vivek Trivedy 撰写。核心公式 **Agent = Model + Harness**——模型包含智能，Harness 让智能变得有用。文章从模型的本体论限制出发（模型只是输入→输出的函数，不会维持状态、执行代码、获取实时信息），反推出 Harness 的六大核心组件：**文件系统**（持久化存储与上下文管理）、**Bash + 代码执行**（通用问题解决）、**沙箱与工具**（安全执行与自我验证循环）、**记忆与搜索**（持续学习）、**上下文管理**（对抗 Context Rot——压缩、工具调用卸载、技能渐进式披露）、**长期自主执行**（Ralph 循环 + 规划 + 自我验证）。文末讨论了模型训练与 Harness 设计的耦合关系：Terminal Bench 2.0 测试显示，仅通过优化 Harness（而非换模型），编码 Agent 排名从全球第 30 升至第 5。结论是**Harness 不仅弥补模型缺陷，更是围绕模型智能构建高效系统——为你的任务优化 Harness 还有很大空间可挖**。

---

**核心事件**：LangChain 作者 Vivek Trivedy 系统定义了 Agent Harness 的概念，从模型的本体论限制推导出六大核心组件，引发开发者社区对 Harness Engineering 的广泛讨论。

### 什么是 Harness？

**Agent = Model + Harness。如果你不是模型，那你就是 Harness。**

Harness 是模型之外的一切代码、配置和执行逻辑。具体包括：系统提示词、工具/技能/MCP、内置基础设施（文件系统/沙箱/浏览器）、编排逻辑（子 Agent 生成/交接/模型路由）、钩子/中间件（压缩/续写/Lint 检查）。

### 六大核心组件

**1. 文件系统**：最基础的 Harness 原语。让 Agent 有工作空间读写数据、增量添加和卸载工作、跨会话保持状态。文件是天然的协作接口，多个 Agent 和人可围绕共享文件协调。Git 提供版本控制。

**2. Bash + 代码执行**：不给一堆工具，给一台"能干活的机器"。模型可以通过代码即时设计自己的工具，不局限于预配置的固定工具集。代码执行已成为自主问题解决的默认通用策略。

**3. 沙箱与工具**：解决安全性和可扩展性。沙箱隔离执行环境、白名单控制命令、强制网络隔离。按需创建、并行展开、用完即销毁。预装语言运行时、Git CLI、浏览器等默认工具，让 Agent 能观察自己的工作结果，形成自我验证循环（写代码→运行→检查日志→修复→再运行）。

**4. 记忆与搜索**：模型除权重和当前上下文外无额外知识。"添加知识"的唯一方式是上下文注入。Harness 支持 AGENTS.md 等记忆文件标准实现持续学习。Web Search 和 Context7 等 MCP 工具获取知识截止日期之后的信息。

**5. 对抗上下文衰减**：当今 Harness 本质上是上下文工程的交付机制。三大手段——**压缩**（上下文快满时智能总结）、**工具调用卸载**（保留头尾 token、完整输出写入文件系统）、**技能渐进式披露**（按需加载，不让工具描述污染上下文）。

**6. 长期自主执行**：文件系统 + Git 跨会话追踪工作。Ralph 循环通过钩子拦截模型退出、在干净上下文中重新注入原始提示词。规划（分解目标为步骤写入文件）+ 自我验证（运行测试→失败反馈→修正）形成稳定闭环。

### 模型训练与 Harness 的耦合

Claude Code、Codex 等产品在训练时将模型和 Harness 同时纳入循环，形成反馈循环但也造成过拟合（如 `apply_patch` 工具逻辑切换时性能下降）。**最适合任务的 Harness 不一定是训练时使用的那个。** Terminal Bench 2.0 实证：仅通过改变 Harness，编码 Agent 从第 30 名升至第 5 名。

### 未来方向

三个开放课题：
- 编排数百个 Agent 在共享代码库中并行工作
- Agent 分析自己的轨迹来识别和修复 Harness 层面的失败模式
- Harness 根据任务即时动态组装合适的工具和上下文

**作者判断**：模型包含智能，Harness 是让智能变得有用的系统。Harness 工程不仅弥补模型缺陷，更是在围绕模型智能构建高效系统。
