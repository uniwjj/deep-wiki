## entry_1.png (720×1320) — Databricks Agent 控制点战略图（核心图）

标题：「Databricks 的 Agent 控制点 — Data + AI Summit 2026 路线观察」
制图：数智技术落地（数栈笔记）

**原来的主场**：Lakehouse / Spark / Delta / MLflow

**新的中枢：企业上下文层** — Genie Ontology / Unity Catalog

六大控制点（自上而下）：
1. **业务入口** — Genie One / Genie Agents / App Builder：把问数、协作、应用和行动推到业务前台
2. **行业场景** — CustomerLake / Profile Agents：把业务入口落到营销和客户运营流程
3. **运行时治理** — Unity AI Gateway / Guardrails：把模型、MCP、工具调用、成本纳入控制面
4. **工程执行面** — Genie Code / Lakeflow / MLflow：让 Agent 进入开发、部署、监控和排障
5. **实时底座** — Lakehouse//RT / Streaming features：为 Agent 提供低延迟上下文和持续反馈
6. **模型生态** — NVIDIA / OpenAI / Anthropic：不押单一模型，押上下文和运行治理

**战略判断**：Databricks 不只想做 AI 数据底座，它在争 Agent 的上下文、工具和业务入口。
