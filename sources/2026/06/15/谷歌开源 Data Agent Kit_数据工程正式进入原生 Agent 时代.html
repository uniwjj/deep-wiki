主要是给数据工程师用的，只不过目前国内云厂商还没有适配，支持 Google Cloud 云原生数据底座（数据仓库 / OLAP、事务型数据库）、开源数据库（MySQL、PostgreSQL、SQL Server、SQLite、OceanBase、TiDB等），它就是一组大数据/数仓相关的Agentic Skills，把原来分散的操作集中在IDE（VS Code、Claude Code、OpenAI Codex CLI）中，直接生成/部署SQL到下游服务中。数据工程师一看就明白了～

发布信息：Google Cloud Tech 2026-06-11 Twitter 官宣开源 Data Agent Kit，一站式整合数据工程 / 科学能力，原生嵌入 VS Code、Claude Code、Codex、Antigravity CLI 等主流 AI 开发环境。

一、Data Agent Kit 到底是什么？

它是谷歌云推出的开源一体化数据智能工具集合，核心定位是给 AI 编程 Agent 预装完整数据工作能力包，而非独立软件。
整体由三大核心模块构成：

  1. 预制数据技能库（Agentic Skills）
    封装标准化数据任务执行逻辑：SQL 查询优化、数据校验、数据漂移检测、dbt 流水线构建、BigQuery/Spark 开发、数据治理、故障排查等，AI 无需重复写底层规范代码。
  2. MCP 模型上下文协议工具层
    安全打通 AI Agent 与谷歌云数据底座（BigQuery、AlloyDB、GCS、Dataproc），免手动编写复杂连接、鉴权、调度代码，一键建立 Agent 与企业数据仓库通路。
  3. 多环境原生插件
    适配主流 AI 代码载体：VS Code 扩展、Claude Code 插件、OpenAI Codex CLI、谷歌自研 Antigravity 终端工具，开箱即用无需改造现有开发环境。

一句话总结：不用切换平台、不用重新学习新工具，你平时写代码的 IDE/AI 终端，直接变身全链路数据工作站。

二、面向谁？精准解决行业三大核心痛点

目标用户群体

  • 数据工程师：搭建数仓、数据流水线、ETL/ELT 开发
  • 数据科学家：特征工程、模型训练、数据质量监控
  • AI 应用开发者：基于企业私有数据构建行业 Agent
  • 数据分析团队：降低 SQL、数据平台上手门槛

当前行业普遍痛点（产品核心解决问题）

  1. 工具割裂，来回切换损耗效率
    现状：写代码用 VS Code、AI 辅助用 Claude Code、查数据登 BigQuery 控制台、调度流水线单独开 CLI，窗口来回切换，上下文频繁丢失。
    解决方案：一套工具全部嵌入现有环境，IDE 内完成数据查询、流水线开发、调试、部署全流程。
  2. AI Agent 不懂数据规范，频繁出现幻觉与错误
    现状：普通 AI 编码工具生成 SQL 无校验、无视数据权限、不做数据漂移检测，产出代码无法直接上生产，人工返工成本极高。
    解决方案：内置强制数据安全规则与行业最佳实践，Agent 自动校验字段、行数、指标分母，禁止无边界查询、原地覆盖数据等危险操作。
  3. 企业数据平台对接复杂，Agent 接入门槛高
    现状：自研 Agent 连接云数仓需要手写鉴权、网络、权限逻辑，中小企业无专职架构师难以落地。
    解决方案：标准化 MCP 协议封装底层连接，一键配置企业数据源，兼顾安全审计与便捷性。

三、三大时代背景

1. AI 编程 Agent 全面普及，数据需求爆发

Claude Code、Codex、Cursor 等本地 AI 开发工具成为标配，但所有工具都缺少专业数据能力，只能生成通用代码，无法适配企业级数据工程，市场存在明显空白。

2. 行业进入 "Agent 原生数据栈" 竞争周期

2026 年各大云厂商全面转向智能体数据架构：谷歌推出 Agentic Data Cloud、Snowflake 发布 Cortex、Databricks 升级 Unity Catalog，核心争夺开发者本地开发入口。
谷歌选择开源免费策略，降低开发者使用门槛，抢占 IDE 端数据 Agent 标准话语权。

3. 打通谷歌完整技术栈，构建差异化壁垒

谷歌拥有三层闭环优势：

  • 底层：BigQuery/Spanner 分布式数据存储
  • 中间：Data Agent Kit 本地 IDE 开发层
  • 上层：Gemini 大模型、ADK 智能体部署框架
    竞争对手（AWS、Azure、Databricks）均无法同时覆盖本地开发环境 + 自研大模型 + 云数据仓库完整链路，本次发布补齐本地开发短板。

五、典型落地使用场景

场景 1：数据工程师流水线极速开发（VS Code）

在 VS Code 安装扩展后，直接用自然语言下达指令：

"基于销售库构建日度用户转化流水线，用 dbt 做分层，写入 BigQuery，添加数据空值、异常值监控，自动生成调度脚本"
Agent 自动完成建表、转换逻辑、质量校验、调度配置，全程无需跳转 BigQuery 控制台。

场景 2：数据科学家本地特征实验（Claude Code）

在 Claude Code 终端中连接企业特征库，让 Agent 自动完成：

  1. 样本数据探查与分布可视化
  2. 特征衍生、缺失值填充、异常过滤
  3. 输出可直接接入 Vertex AI 训练的数据集脚本

场景 3：AI 应用开发者构建私有数据 Agent（Antigravity CLI/Codex）

本地 CLI 环境下，为业务智能体赋予数据查询能力，业务人员自然语言提问即可读取脱敏企业经营数据，自动生成合规报表，无需数据团队介入。

场景 4：中小企业轻量化数据治理

无专职数据架构团队，依靠预制技能库统一 SQL 规范、数据权限、变更日志，低成本落地企业数据治理流程。

七、长期影响：将重塑数据从业者工作方式与市场格局

1. 用户行为变革：数据工作彻底从 "平台端迁移到本地 IDE"

过去数据工作 80% 操作在云端控制台；未来开发者 90% 数据任务在日常代码编辑器完成，云端仅负责存储与调度。
数据工程师、科学家不再需要频繁切换网页后台，工作流与通用软件开发完全统一。

2. 岗位能力重构：人人可做轻量化数据开发

业务分析师、AI 应用开发者无需精通 BigQuery、dbt、Spark 底层语法，依靠 Agent + 预制技能完成常规数据需求，数据团队从基础取数、流水线搭建中解放，专注复杂架构与深度建模。

3. 行业市场格局变化

  1. 谷歌补齐本地开发短板：过去谷歌云优势集中在云端数据仓库，本地开发工具薄弱，本次开源后形成 "本地 IDE 开发 + 云端数据底座 + 自研大模型" 完整闭环，大幅提升对数据团队的吸引力。
  2. 定义 Agent 数据开发标准：开源 + 跨工具兼容策略，推动 MCP 协议、数据 Agent 技能库成为行业通用规范，挤压封闭型私有数据工具生存空间。
  3. 倒逼竞品跟进本地 IDE 生态：Snowflake、Databricks、AWS 将加速推出 VS Code、Claude Code 原生插件，本地 AI 开发环境成为云厂商必争赛道。

4. 降低企业数据 Agent 落地成本

此前企业搭建私有数据智能体，需投入大量人力开发连接、校验、权限逻辑；Data Agent Kit 提供成熟开源底座，中小企业可低成本落地数据驱动 AI 应用，加速企业 Agent 普及。

八、总结：事件核心价值一句话概括

谷歌开源 Data Agent Kit 的本质，是把企业级数据工程能力下沉到开发者日常使用的 AI 编程环境，以开源中立、全工具兼容的策略抢占本地数据 Agent 标准，完成从云端数仓到终端开发的全链路生态闭环，正式宣告 "IDE 原生数据智能体" 时代到来。

附录：国内云产品支持情况（阿里云 / 腾讯云 / 华为云 / 火山引擎）

结论基本不可用

  • 无官方预制连接器：Google Data Agent Kit 没有针对国内云厂商（阿里、腾讯、华为、火山）做专属适配、一键鉴权、原生权限打通；
  • 可通过通用协议兼容接入（技术可行）：国内云数仓底层兼容标准 MySQL/PostgreSQL/Trino SQL，依靠 MCP 通用数据库驱动即可连接；
  • 存在明显使用限制：缺少云厂商特有能力封装，部分云特有语法、鉴权、存储过程无法自动生成。

国内主流云产品接入可行性分档

【可直接连接，基础 SQL 可用】兼容 PostgreSQL/MySQL 协议的国内 OLAP

阿里云 Hologres（PostgreSQL 兼容，最顺滑） AnalyticDB PostgreSQL MaxCompute（通过 Trino/ODPS JDBC 驱动连接，语法需人工微调） RDS MySQL、PolarDB

腾讯云 TDSQL-C PostgreSQL/MySQL Doris（TCHouse-D，兼容 MySQL 协议）

华为云 GaussDB PostgreSQL DWS 数据仓库

火山引擎 ByteHouse（ClickHouse 兼容） VeDB（MySQL/PostgreSQL）

【有较大适配成本】国内专有大数据平台

阿里云 MaxCompute、腾讯云 WeData、华为云 DLI： 原生语法、UDF、调度体系与 BigQuery 差异大，Data Agent Kit 自动生成代码会大量报错，需要二次改写 MCP 技能库做语法翻译。

国内云接入的 3 个核心痛点

  • 鉴权不原生 BigQuery 支持 Google Service Account 一键授权；国内云需手动配置 AK/SK、VPC 白名单、SSL 证书，无可视化配置面板。
  • 云特有能力无法自动生成 如 MaxCompute 的 Tunnel、Hologres 分区表语法、ByteHouse 自研函数，Agent 仅生成标准 SQL，无法调用国内云专属特性。
  • 无官方数据治理打通 无法自动读取阿里云 DataWorks、腾讯 WeData 的数据目录、数据标准、血缘，元数据探查能力大幅削弱。
