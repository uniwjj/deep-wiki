分层抽象架构设计

Gravitino 采用经典的分层架构设计，将复杂性问题分解为多个可管理的层次，每一层都有明确的职责边界：

功能层提供统一的元数据管理和治理能力，包括标准的元数据创建、更新、删除操作，以及统一的访问控制、审计和发现功能。这一层确保了无论底层数据源如何变化，上层应用都能获得一致的体验。

接口层通过 REST API 提供标准化访问接口，未来还将支持 Thrift 和 JDBC 接口。这种多协议支持确保了 Gravitino 能够与各种数据处理引擎和工具无缝集成。

核心对象模型定义了通用的元数据模型，能够表示不同来源和类型的元数据。这是 Gravitino 架构的核心创新，它抽象出了统一的元数据表示方式：

-- 跨多个数据源的联邦查询
SELECT 
    o.order_id,
    o.customer_id,
    c.customer_name,
    p.product_name,
    k.click_event_time
FROM mysql_catalog.sales.orders o
JOIN postgresql_catalog.customers.customer_info c 
    ON o.customer_id = c.customer_id
JOIN iceberg_catalog.products.product_catalog p 
    ON o.product_id = p.product_id
LEFT JOIN kafka_catalog.user_events.click_events k 
    ON o.customer_id = k.user_id
WHERE o.order_date >= CURRENT_DATE - INTERVAL '7' DAY;
